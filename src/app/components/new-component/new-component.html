Componentes

Um componente é a unidade básica de UI do Angular: combina um template HTML, estilos CSS e lógica TypeScript em um bloco encapsulado e reutilizável, identificado por um seletor customizado (ex: <app-header>).

Text interpolation

É uma maneira fácil e poderosa de exibir valore dinâmicos em seu template HTML.

Ele permite que você insira valores variáveis diretamente no seu código HTML tornando a exibição de dados dinamicos muito simples.

Binding

É um coneito fundamento que se refere à maneira como os dados fluem entre a parte lógica de um aplicativo e a interface do usuário.

O binding permite conectar os dados da aplicação com os elemento do template HTM, tornando-o din6amico e interativo

Existem cinco tipos de binding:

  • Property Binding
  • Attribute Binding
  • Class and style Binding
  • Event Binding
  • Two-way Binding

Template Reference Variables

São uma maneira útil de acessar elemento do ser template HTML diretamente no código.

Eles sao definidos nos elementos HTML com o uso do símbolo # seguido por um nome de variável.

Isso perque que haja a comunicação entre o compornente e o template de uma forma direta.


Exsitem algumas formas de de uso:

  • #nome
  • viewChild('nome') nome: ElemntRef

IMPORTANTE: As variáveis só devem ser usadas quando não tiver uma alternativa mais adequada

Control Flow

Control Flow é a sintaxe nativa do Angular (introduzida na v17) para controle de renderização diretamente no template, substituindo as diretivas estruturais (*ngIf, *ngFor, *ngSwitch) por blocos declarativos: 


          @if (condition) { ... } @else { ... }
          @for (item of list; track item.id) { ... }
          @switch (value) { @case (x) { ... } }

Deferrable Views

Deferrable Views é o mecanismo do Angular para carregar blocos de template de forma lazy (adiada), controlando quando e sob quais condições o conteúdo é renderizado e o JavaScript correspondente é baixado: 


        @defer (on viewport) {
           <app-heavy-component />
        } @placeholder {
          <p>Carregando...</p>
        } @loading {
          <spinner />
        } @error {
          <p>Falhou</p>
        }

Os gatilhos (on viewport, on idle, on interaction, when condition) determinam o momento do carregamento, reduzindo o bundle inicial e melhorando o LCP/TTI.
