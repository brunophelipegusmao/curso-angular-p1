Control Flow

Control Flow é a sintaxe nativa do Angular para controlar a renderização do template de forma declarativa, usando os blocos @if, @for e @switch diretamente no HTML — substituindo definitivamente as antigas diretivas estruturais *ngIf, *ngFor e *ngSwitch.

IF

TypeScript:

       export class AppComponent {
          isLogado = signal(true);
    }

HTML:

            @if (usuario$ | async; as usuario) {
  @if (usuario.logado) {
    <p>Bem-vindo, {{ usuario.nome }}!</p>
  } @else {
    <p>Acesso negado.</p>
  }
} @else {
  <p>Carregando...</p>
}
Resultado do exemplo: @if (isLogado) {

Bem-vindo!

} @else {

Acesso negado.

}

A sintaxe moderna com @if foi introduzida no Angular 17. No Angular 21 ela já é o padrão — o *ngIf é considerado legado.

Apenas a título de ilustração está abaixo como era feito o mesmo if do exeplo acima, afirmando apenas que no TypeScript as declarações são as mesmas nos dois casos. 

          <p *ngIf="isLogado; else pendente">Bem-vindo!</p>
    <ng-template #pendente>
        <p>Acesso negado.</p>
    </ng-template>

Essa mudança trouxe melhorias em performace e em leitura de código.

FOR

TypeScript:

          export class AppComponent {
    usuarios = signal([
        { id: 1, nome: 'Ada' },
        { id: 2, nome: 'Alan' },
        { id: 3, nome: 'Grace' },
      ]);
    }

Nova sintaxe (Angular 17+):

          @for (usuario of (usuarios$ | async) ?? []; track usuario.id) {
      <p>{{ usuario.nome }}</p>
    } @empty {
      <p>Nenhum usuário encontrado.</p>
    }

Forma antiga (legado):

    <p *ngFor="let usuario of usuarios">{{ usuario.nome }}</p>

O track é obrigatório na nova sintaxe — equivale ao trackBy do *ngFor e otimiza a renderização da lista. O bloco @empty não tinha equivalente direto no *ngFor.

Apenas a título de ilustração está abaixo como era feito o mesmo if do exeplo acima, afirmando apenas que no TypeScript as declarações são as mesmas nos dois casos.

@for (usuario of (usuario$ | async) ?? []; track usuario.id) {

{{ usuario.nome }}

} @empty {

Nenhum usuário encontrado.

}

Observações:

  • ?? [] no @for — obrigatório pois o async pipe retorna null antes do Observable emitir, e o @for não aceita null.
  • as usuario no @if — o padrão recomendado pela documentação do Angular para trabalhar com Observables no template, evita múltiplas subscrições.
  • AsyncPipe no imports — no lugar do CommonModule inteiro, importe apenas o AsyncPipe — é a abordagem correta em componentes standalone no Angular 21.

For e seus valores

Track

Aqui devemos destacar o track que possui alguns valores que merecem atenção:

O track é obrigatório no @for e indica ao Angular qual propriedade usar para identificar cada item da lista de forma única. Com isso, quando a lista muda, o Angular reutiliza os elementos DOM existentes ao invés de recriar tudo — tornando a renderização mais eficiente: 

        @for (usuario of usuarios; track usuario.id) {
  <p>{{ usuario.nome }}</p>
}

É o equivalente ao trackBy do antigo *ngFor, porém agora é obrigatório — o Angular 21 exige sua declaração explícita. Use sempre uma propriedade única como id. Caso o item não tenha identificador, use track $index como alternativa.

O @for disponibiliza variáveis implícitas que podem ser usadas diretamente no template ou aliasadas com let: 

        @for (usuario of usuarios; track usuario.id; let i = $index, e = $even) {
  <p>{{ i }} - {{ usuario.nome }}</p>
}
  • $index — posição atual do item na lista (começa em 0)
  • $count — total de itens na lista
  • $first — true se for o primeiro item
  • $last — true se for o último item
  • $even — true se o índice for par
  • $odd — true se o índice for ímpar

São especialmente úteis em listas com estilos alternados, numeração visível ou comportamentos distintos para o primeiro e último elemento.

For - empty

O @empty é um bloco opcional do @for que é renderizado automaticamente quando a lista estiver vazia ou nula — eliminando a necessidade de um @if adicional para tratar esse caso: 

        @for (usuario of usuarios; track usuario.id) {
  <p>{{ usuario.nome }}</p>
} @empty {
  <p>Nenhum usuário encontrado.</p>
}

Não tinha equivalente direto no antigo *ngFor — era necessário combinar com um *ngIf separado para obter o mesmo comportamento.
